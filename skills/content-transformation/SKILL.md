# Content Transformation

## Instructions

### Goal

- Receber qualquer conteúdo-fonte (artigo, relatório, apresentação, post, e-mail, documento interno) e transformá-lo em um ou mais formatos de marketing otimizados para canais específicos da Agio.
- Preservar mensagens-chave e dados de prova enquanto recalibra tom, estrutura e vocabulário para o canal de destino.
- Garantir que toda peça transformada esteja ancorada nas 5 Narrativas Agio e siga a regra de tom 60/40 (acessível/técnico).

### Required Inputs

Confirmar cada item com o usuário. Perguntar um de cada vez se estiver faltando — nunca prosseguir com informação incompleta.

1. **Conteúdo-fonte** — O conteúdo original a ser transformado (texto colado, resumo de documento ou link).
2. **Formato(s) de destino** — Post LinkedIn, carrossel Instagram, e-mail nurture, seção de landing page, blurb de newsletter, mensagem WhatsApp, slide de apresentação, etc.
3. **Canal principal** — Canal onde o conteúdo será publicado.
4. **Persona-alvo** — Gestor de Facilities, Arquiteto, C-Level ou público geral.
5. **Objetivo da peça** — O que o leitor deve fazer após consumir (awareness, consideração, conversão, retenção).
6. **Contexto adicional (opcional)** — Nome de campanha, data de lançamento, links complementares, referências visuais.

### Workflow

1. **Receber e analisar conteúdo-fonte.** Ler o material completo. Identificar formato atual (relatório, artigo, apresentação, post, memo interno), extensão, tom e registro de linguagem.

2. **Extrair mensagens-chave.** Isolar as 3–5 mensagens centrais, dados de prova e proof points. Sinalizar estatísticas (ex.: NPS 92.7%, 320+ projetos), nomes de clientes, nomes de projetos e termos técnicos que devem ser preservados.

3. **Mapear para Narrativas Agio.** Para cada mensagem-chave, identificar qual das 5 Narrativas sustenta:
   - Caos → Previsibilidade
   - Complexidade → Simplicidade
   - Problema → Solução
   - Reativo → Proativo
   - Genérico → Personalizado

   Se uma mensagem não se encaixa em nenhuma narrativa, sinalizar para o usuário — pode precisar de reframing ou exclusão.

4. **Selecionar formato(s) de destino via Matriz de Transformação.**

   | Fonte | → LinkedIn Post | → Carrossel IG | → E-mail | → Landing Page | → Newsletter | → WhatsApp |
   |---|---|---|---|---|---|---|
   | Relatório interno | Destaque + insight | Dados em slides | Achado-chave + CTA | Seção de prova | Resumo + link | Teaser 1 linha |
   | Artigo/blog | Thread ou post único | Pontos-chave visuais | Trecho + link | Seção hero | Item do digest | Snippet compartilhável |
   | Case study | História antes/depois | 5-7 slides narrativos | Sequência drip | Bloco testimonial | Mini-case | Resultado destaque |
   | Apresentação | Insight de slide-chave | Adaptação de slides | E-mail resumo | Seção de feature | Lista de takeaways | Muito longo |
   | Post social | Expandir/reformatar | Adaptar cross-canal | Contexto + profundidade | Prova social | Item curado | Compartilhar direto |

5. **Recalibrar tom para o canal de destino.**
   - **LinkedIn:** Analítico, B2B, registro de thought-leadership. Parágrafos mais longos OK. Data-driven. Falar com decisores. Usar 1ª pessoa plural ("Na Agio, entregamos...").
   - **Instagram:** Visual-first, conciso, escaneável. Frases curtas. Emoji com moderação (max 3 por post). Slides de carrossel: 1 ideia por slide, max 15 palavras por slide.
   - **E-mail:** Conversacional, direto, personalizado. Assunto claro. CTA único. Tom profissional mas próximo.
   - **Landing page:** Headlines orientadas a benefício. Seções escaneáveis. Múltiplos CTAs. Prova social proeminente.
   - **Newsletter:** Tom curado, resumos breves, orientado a links. Valor primeiro.
   - **WhatsApp:** Ultra-conciso, 2-3 frases max. Informal mas on-brand.

   Aplicar a regra 60/40 em cada output: 60% linguagem acessível que um não-engenheiro entende, 40% termos de credibilidade técnica (Turnkey, retrofit, comissionamento, Método Agio).

6. **Aplicar vocabulário obrigatório.** Varrer o rascunho para termos proibidos e substituir:
   - "construtora" → "empresa de Soluções Personalizadas de Engenharia"
   - "obra" (genérico) → "projeto", "empreendimento" ou "operação"
   - "excelência" → remover ou substituir por prova específica (ex.: "NPS 92.7%")
   - "inovador" → substituir por diferencial concreto (ex.: "modelo Turnkey com gestão integrada")
   - "revolucionário" → substituir por resultado mensurável

   Injetar termos âncora Agio onde natural: "Soluções Personalizadas de Engenharia", "Turnkey", "Previsibilidade", "Método Agio".

7. **Adaptar estrutura por formato.**
   - **Post LinkedIn:** Hook (1ª linha captura atenção) → Contexto (2-3 frases) → Insight ou dado → Takeaway → CTA. Max 1300 caracteres para visibilidade no feed; até 3000 para thought-leadership.
   - **Carrossel Instagram:** Slide 1 = hook/título → Slides 2-6 = um ponto-chave cada → Slide final = CTA + marca. Descrições de alt-text para cada slide.
   - **E-mail:** Assunto (max 50 chars) → Preview text → Hook de abertura → Corpo (2-3 parágrafos curtos) → Texto de CTA único → Linha PS (opcional).
   - **Seção Landing page:** Headline → Subheadline → 3-4 bullets de benefício → Quote de prova social → CTA.
   - **Blurb Newsletter:** Headline em negrito → Resumo de 2 frases → "Leia mais →" link.
   - **WhatsApp:** Saudação → 1 frase de value prop → Link ou CTA.

8. **Gerar CTA alinhado ao objetivo.**
   - Awareness: "Saiba como a Agio entrega previsibilidade em projetos complexos"
   - Consideração: "Veja como o Método Agio reduziu [X]% no prazo do projeto [Nome]"
   - Conversão: "Fale com nosso time → [link]"
   - Retenção: "Confira os resultados do seu projeto no relatório completo"

   O CTA deve sempre conectar a um arco narrativo — nunca ser genérico ("Saiba mais" sozinho é fraco — sempre adicionar o benefício).

9. **Adaptar para persona.**
   - **Gestor de Facilities:** Enfatizar continuidade operacional, zero downtime, cronogramas previsíveis. Termos: "operação assistida", "comissionamento", "manutenção preditiva".
   - **Arquiteto:** Enfatizar fidelidade ao projeto, colaboração técnica, especificações de materiais. Termos: "memorial descritivo", "compatibilização", "detalhamento executivo".
   - **C-Level:** Enfatizar ROI, mitigação de risco, valor estratégico. Termos: "previsibilidade orçamentária", "gestão integrada", "modelo Turnkey".

10. **Verificar pilar de conteúdo.** Tagar o output com seu pilar:
    - Técnico/Educacional — explica um conceito ou método
    - Cases/Prova Social — mostra resultado de projeto real
    - Institucional — reforça identidade de marca ou cultura
    - Mercado — comenta tendências do setor
    - Comercial — direciona uma ação de negócio

11. **Formatar e entregar.** Apresentar cada peça transformada com:
    - Label de formato e canal
    - Tag de pilar de conteúdo
    - Tag de persona
    - Contagem de caracteres/palavras
    - O conteúdo transformado
    - Hashtags sugeridas (para social) ou linha de assunto (para e-mail)

12. **Oferecer variações.** Após entregar o output principal, oferecer proativamente:
    - Versão mais curta (se o output excede normas do canal)
    - Ângulo alternativo usando uma Narrativa Agio diferente
    - Adaptação para persona secundária

### Anti-patterns

- **Nunca copiar e colar com edições superficiais.** Cada output deve ser estrutural e tonalmente reconstruído para seu formato de destino. Mudar poucas palavras não é transformação.
- **Nunca usar vocabulário proibido:** "construtora", "obra" (genérico), "excelência", "inovador", "revolucionário". Devem ser capturados e substituídos em todo output.
- **Nunca produzir CTA que diga apenas "Saiba mais" ou "Clique aqui."** Todo CTA deve incluir benefício ou resultado vinculado a uma narrativa Agio.
- **Nunca ignorar a regra de tom 60/40.** Se um output lê como paper acadêmico (técnico demais) ou blog lifestyle (casual demais), recalibrar.
- **Nunca remover dados e proof points do fonte.** Números, NPS, nomes de projetos e resultados mensuráveis são os ativos mais fortes da Agio — devem sobreviver à transformação.
- **Nunca produzir texto de carrossel Instagram com mais de 15 palavras por slide.** Carrosséis são visual-first; texto denso anula o propósito.
- **Nunca entregar sem especificar pilar de conteúdo e persona.** Tags obrigatórias para o calendário de conteúdo da Agio.
- **Nunca inventar dados de projeto ou nomes de clientes.** Usar apenas fatos presentes no conteúdo-fonte. Se uma stat é necessária mas ausente, sinalizar para o usuário.

### Quality Gate

Antes de entregar, verificar cada item:

- [ ] Todos os termos proibidos eliminados ("construtora", "obra" genérico, "excelência", "inovador", "revolucionário")
- [ ] Pelo menos um termo âncora Agio presente ("Soluções Personalizadas de Engenharia", "Turnkey", "Previsibilidade", "Método Agio")
- [ ] Tom segue a regra 60/40 (acessível/técnico)
- [ ] Dados-chave e provas do fonte preservados
- [ ] CTA inclui benefício, não apenas "Saiba mais"
- [ ] Pilar de conteúdo e persona tagados
- [ ] Contagem de caracteres/palavras dentro das normas do canal
- [ ] Nenhum dado inventado ou nomes de clientes fabricados
- [ ] Pelo menos uma Narrativa Agio explicitamente ancorada
- [ ] Estrutura do formato corresponde às convenções do canal (hook → corpo → CTA)
